# 视觉系小说 - 项目扩展优化指南

## 项目概述

本项目是一个基于UE5 C++的视觉系小说游戏，采用数据驱动架构，所有剧情内容、美术资源均从外部文件加载，支持高扩展性和易替换性。

## 二、剧情内容修改接口

### 2.1 修改剧情文本

**文件位置**: `StoryData/main_story.json`

**修改方式**: 直接编辑JSON文件

**JSON结构说明**:

```json
{
  "StoryId": "main_story",        // 剧情唯一ID，新增剧情时需更改
  "Title": "晨曦之约",            // 剧情标题
  "FirstNodeId": "prologue_01",   // 第一个剧情节点ID
  "Nodes": {
    "节点ID": {
      "Id": "节点ID",
      "Type": "Dialogue",          // 类型: "Dialogue"(对话) 或 "Choice"(选择)
      "Speaker": "角色名",          // 说话者名称，旁白留空
      "Text": "对话内容",           // 显示的文本
      "Background": "bg_id",       // 背景图片ID（对应Backgrounds文件夹中的文件名）
      "Characters": [              // 出场角色列表
        {
          "Id": "角色ID",           // 对应Characters文件夹中的文件名前缀
          "Position": "Center",     // 位置: Left/Center/Right/CenterLeft/CenterRight
          "Expression": "normal"    // 表情: 对应立绘文件名后缀
        }
      ],
      "Next": "下一节点ID",         // 下一个节点（Dialogue类型）
      "Choices": [                 // 选项列表（Choice类型）
        {
          "Id": "选项ID",
          "Text": "选项文本",
          "Next": "分支节点ID"
        }
      ],
      "Music": "音乐ID"            // 可选：背景音乐ID
    }
  }
}
```

### 2.2 新增剧情章节

1. 在 `StoryData/` 目录下创建新的JSON文件，如 `chapter2.json`
2. 按照上述JSON结构编写新章节内容
3. 修改 `VNGameInstance.h` 中的 `DefaultStoryId` 或通过代码切换剧情

**代码位置**: `Source/VisualNovel/Core/VNGameInstance.h`
```cpp
UPROPERTY(EditDefaultsOnly, Category = "VisualNovel")
FString DefaultStoryId = TEXT("main_story");  // 修改此处切换默认剧情
```

### 2.3 新增选择节点

在JSON的Nodes中添加Type为"Choice"的节点：

```json
"choice_new_01": {
  "Id": "choice_new_01",
  "Type": "Choice",
  "Speaker": "",
  "Text": "你要怎么做？",
  "Background": "bg_current",
  "Characters": [],
  "Choices": [
    { "Id": "opt_a", "Text": "选项A", "Next": "branch_a_01" },
    { "Id": "opt_b", "Text": "选项B", "Next": "branch_b_01" },
    { "Id": "opt_c", "Text": "选项C", "Next": "branch_c_01" }
  ]
}
```

## 三、美术资源替换接口

### 3.1 背景图片

**存放目录**: `Resources/Backgrounds/`

**文件命名规则**: `{BackgroundId}.{扩展名}`
- BackgroundId 必须与JSON剧情文件中的 `Background` 字段一致
- 支持的格式: `.png`, `.jpg`, `.jpeg`, `.bmp`, `.hdr`
- 推荐分辨率: 1920x1080 或更高
- 推荐格式: `.jpg`（照片类背景）或 `.png`（需要透明通道时）

**替换步骤**:
1. 准备新背景图片，命名为与JSON中Background字段相同的名称
2. 将文件放入 `Resources/Backgrounds/` 目录
3. 重新启动游戏，资源加载器会自动加载新图片

**当前背景资源列表**:
| BackgroundId | 文件名 | 描述 |
|---|---|---|
| bg_room_morning | bg_room_morning.jpg | 早晨房间 |
| bg_street_morning | bg_street_morning.jpg | 清晨街道 |
| bg_bus | bg_bus.jpg | 公交车内 |
| bg_school_gate | bg_school_gate.jpg | 学校大门 |
| bg_classroom | bg_classroom.jpg | 教室 |
| bg_rooftop | bg_rooftop.jpg | 天台 |
| bg_library | bg_library.jpg | 图书馆 |
| main_menu | main_menu.jpg | 主菜单背景 |

### 3.2 人物立绘

**存放目录**: `Resources/Characters/`

**文件命名规则**: `{CharacterId}_{Expression}.{扩展名}`
- CharacterId 与JSON中的 `Characters[].Id` 一致
- Expression 与JSON中的 `Characters[].Expression` 一致
- 支持的格式: `.png`（推荐，支持透明背景）, `.jpg`
- 推荐尺寸: 宽度600-800px，高度900-1200px
- **强烈推荐使用PNG格式带透明背景**，以实现立绘叠加效果

**替换步骤**:
1. 准备新立绘图片，命名为 `{角色ID}_{表情}.png`
2. 将文件放入 `Resources/Characters/` 目录
3. 如果没有特定表情的立绘，加载器会自动回退到 `{CharacterId}.png`

**当前角色资源列表**:
| CharacterId | Expression | 文件名 |
|---|---|---|
| protagonist | normal | protagonist_normal.jpg |
| heroine | normal | heroine_normal.jpg |
| heroine | smile | heroine_smile.jpg |
| friend_male | normal | friend_male_normal.jpg |

**新增角色**:
1. 在 `Resources/Characters/` 中添加立绘文件
2. 在剧情JSON的 `Characters` 数组中引用新的角色ID
3. 无需修改任何C++代码

### 3.3 音频资源（预留接口）

**存放目录**: `Resources/Audio/`

**支持的格式**: `.wav`, `.mp3`, `.ogg`

**命名规则**: `{MusicId}.{扩展名}` 或 `{SFXId}.{扩展名}`

> 注：音频播放功能尚未实现，但目录结构和数据接口已预留。后续添加迷你音乐播放器模块时可直接使用。

## 七、编译与运行

### 7.1 首次编译

1. 右键点击 `VisualNovel.uproject` → 选择 **"Generate Visual Studio project files"**
2. 打开生成的 `VisualNovel.sln`
3. 在Visual Studio中选择 **"Development Editor"** 配置
4. 按 **Ctrl+Shift+B** 编译
5. 编译成功后，双击 `VisualNovel.uproject` 打开UE编辑器

### 7.2 在UE编辑器中运行

1. 打开项目后，UE编辑器会自动加载
2. 点击工具栏的 **Play** 按钮运行游戏
3. 首次运行需要创建一个空地图（Level），保存为 `/Game/Maps/MainMenu`

### 7.3 常见编译问题

| 问题 | 解决方案 |
|---|---|
| 找不到.generated.h文件 | 右键.uproject → Generate Visual Studio project files |
| 链接错误 | 清理项目（Clean Solution）后重新编译 |
| 模块未找到 | 检查 Build.cs 中的模块名是否正确 |
| 中文路径问题 | 确保项目路径不含特殊字符 |

## 九、架构设计说明

### 9.1 核心设计原则

- **数据驱动**: 所有剧情内容存储在JSON文件中，无需修改代码即可更新内容
- **运行时资源加载**: 图片从文件系统动态加载，无需在UE编辑器中导入
- **模块解耦**: 各系统通过委托(Delegate)通信，互不直接依赖
- **高内聚低耦合**: 每个模块职责单一，接口清晰

### 9.2 数据流

```
玩家输入 → VNPlayerController → VNGameMode → VNStoryEngine → 委托广播
                                                            ↓
                                                     VNGameMode接收
                                                            ↓
                                              VNAssetLoader(加载图片)
                                              VNGameWidget(更新UI)
                                              VNSaveManager(记录进度)
```

### 9.3 存档机制

- 存档保存位置: `{ProjectSavedDir}/SaveGames/`
- 存档格式: JSON
- 存档内容: 当前节点ID、选择历史、访问过的节点列表
- 读取存档时从最后一个选择节点之后的首幕剧情开始
